# Volvo On Call CN v3.2.0

本版本面向长期无人值守运行场景，集中修复安全性与稳定性问题。建议所有用户升级。

## 安全修复（重要）

- **修复 refresh token 明文泄漏进日志的问题。** 旧版本在 HTTP 请求失败时以 `exc_info` 输出完整异常堆栈，堆栈中的请求 URL 携带未脱敏的 refresh token（日志消息本身脱敏了，但堆栈没有）。新版本对 401/403/4xx 失败统一抛出脱敏后的异常副本，不再输出携带 token 的堆栈。
  - **建议：升级后在沃尔沃 App 中修改一次密码**，使历史日志中可能已泄漏的 token 全部失效；分享旧日志前请自行检查。
- 新增重新认证（reauth）流程：密码被服务器连续拒绝时，Home Assistant 会弹出"需要重新认证"通知并引导重新输入密码，不再无限静默失败。

## 稳定性修复

- **gRPC 全面迁移到 grpc.aio 异步通道。** 旧版本在事件循环中直接执行同步 gRPC 调用，网络劣化时单次调用最多阻塞整个 Home Assistant 10 秒（每辆车每轮 8-9 个调用会连环阻塞）。迁移后所有车辆数据源真正并发拉取，不再阻塞 UI 和其它自动化。
- **会话被服务器提前作废（401/403）时立即恢复。** 旧版本只会在 token 临近到期时才刷新，实测会僵死约 19 分钟；新版本在收到 401/403 时立即标记 token 失效，并在同一轮更新内完成"刷新 token → 密码重登录"恢复链路。
- **4xx 确定性错误不再重试**（429 除外），避免用失效 token 高频轰炸 API 加重服务端风控；5xx/网络错误保留指数退避重试，并移除了旧的双层嵌套重试。
- **密码登录本地限流**：两次密码登录之间强制间隔 120 秒，防止故障期间高频密码登录触发账号风控。
- **登录会话跨重启持久化**：refresh token 保存在 Home Assistant `.storage`（私有权限文件，键名为账号哈希），重启后优先用 refresh token 续期而非重新密码登录。
- **Vehicle 对象跨轮询复用，单源故障回退真正生效。** 旧版本每轮重建 Vehicle，导致"单个数据源失败时回退上次数据"的缓存永远为空，某个 gRPC 源失败会让实体显示构造默认值（例如锁显示为未锁、里程为 0、位置为 (0,0)）。现在缓存跨轮次生效，单源故障回退到最近一次真实数据。
- **失败有界降级**：更新连续失败最多 5 轮内继续提供缓存数据，超过后实体如实变为不可用（unavailable），避免自动化长期基于过期状态误动作；恢复后自动回归正常。
- 每辆车的持久化 Store 只创建和读盘一次并复用，不再每 30 秒读一次磁盘。
- 配置选项变更时仅在账号密码变化时重建 API 客户端，并正确关闭旧的 gRPC 通道；集成卸载时同样释放通道。

## 弃用 API 清理（预防未来 HA 升级破坏）

- `entity_id` 中的 VIN 统一小写化（slugify），消除 "invalid entity ID" 警告；该警告对应的行为将在 Home Assistant 2027.2 起变为硬错误。
- `device_tracker` 改用新的 `TrackerEntity` 导入路径（旧别名 2027.6 移除）。
- 移除 `homeassistant.const` 通配符导入，消除 `CONCENTRATION_PARTS_PER_CUBIC_METER` 弃用警告（2027.7 移除）。
- 实体改为按 VIN 定位车辆数据，不再依赖车辆列表顺序，多车账号列表顺序变化不会再错位。

## 升级注意事项（entity_id）

- 本版本起集成向 Home Assistant 申报的 `entity_id` 由「大写 VIN」改为「小写 VIN」（例如 `sensor.LVY..._odometer` → `sensor.lvy..._odometer`）。
- **绝大多数用户无需任何操作**：已注册实体的最终 ID 由 Home Assistant 实体注册表决定，注册表中保存的本来就是小写形式，升级后实体 ID 不变。
- 如果你曾手工改过实体 ID 或在自动化/卡片中硬编码了大写 VIN 的实体 ID，升级后请到 设置 → 设备与服务 → 实体 中核对实际 ID 并同步修改引用。

- 全量测试套件 98 项通过（含覆盖率门槛）。
