# Volvo On Call CN v3.0.2

本版本把 2.7 的 T8 行程和电池能力扩展为可直接用于 Home Assistant 的原生统计体验，并加入按 Volvo Cars App 视觉逻辑重构的 Lovelace 卡片。

## Volvo 原生风格统计卡

- 新增 `custom:volvo-car-card`，随集成提供，不需要单独安装前端仓库。
- 卡片按 Home Assistant Sections 12 列网格适配，移动端优先。
- 参考 `volvo_ui_study` 的 DLS token、俯视车辆状态层和 TM/TA 统计结构重构。
- 首屏展示车辆身份、连接状态、锁状态、纯电续航、燃油续航、电量、油量、充电功率和预计充满时间。
- 俯视车辆图叠加四门、四窗、引擎盖、尾门、天窗状态。
- 行程统计分组显示 TM 和 TA；T8 可显示 TM 电耗。
- 支持锁车/解锁、远程启动、尾门、天窗、闪灯、鸣笛闪灯控制；危险操作二次确认。

## 原生长期统计

- 新增 `sensor.{vin}_full_charge_electric_range`。
- T8 电量首次达到 `100%` 时记录一次当次纯电续航；同一次满电停留不会重复采样。
- 电量低于 `100%` 后自动准备下一次满电采样。
- 样本写入 Home Assistant 存储，重启或重载后保留。
- 传感器使用距离设备类型和 `measurement` 状态类，可直接接入 Recorder 与 `statistics-graph`。

## 本地素材边界

- 仓库不分发 Volvo App 的专有车辆图片、字体或 APK。
- 可用 `[removed legacy local-asset helper] /path/to/base.apk` 在本机提取俯视车图。
- 提取出的 `custom_components/volvooncall_cn/frontend/cartopview_complete_fallback.png` 已被 `.gitignore` 排除。
- 卡片自定义图片只接受同源路径或 HTTPS URL，拒绝 `http://` 外部图片。

## 稳定性和安全修复

- 定时刷新和操作后手动刷新统一受协调器扫描间隔节流，最小扫描间隔为 30 秒。
- 数字沃尔沃 HTTP 请求失败日志会脱敏 refresh token、Bearer token 和 X-Token。

## 验证

- Python 单元测试覆盖满电采样、动力类型过滤、TM/TA 元数据和现有协调器行为。
- Node 前端测试覆盖卡片注册、实体推导、统计建议、危险操作确认和充电状态判断。
- 定时刷新和操作后手动刷新统一受协调器扫描间隔节流，最小扫描间隔为 30 秒。
